4. Log Routing 로그 라우팅 (레이블/네임스페이스 기반)

Namespace Routing

production/* → ES hot tier (7일 보관)
staging/* → ES warm tier (3일)
kube-system/* → Loki (1일)

Label-based Routing

app=api-gateway → 전용 인덱스
level=ERROR → alert-logs 토픽
audit=true → 감사 로그 장기 보관

Fluent Bit Config

[FILTER] grep 필터링
[FILTER] parser JSON 파싱
[FILTER] modify 필드 추가
[OUTPUT] match tag.* 라우팅
