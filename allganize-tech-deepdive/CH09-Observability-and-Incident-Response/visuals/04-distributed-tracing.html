4. Sampling Strategies 샘플링 전략

Head-based Sampling (헤드 기반)

요청 시작 시점에 샘플링 결정. 첫 번째 서비스에서 결정하면 이후 모든 서비스가 동일하게 따름.
// 요청 시작 시 결정
if random() < sampling_rate:
    trace.sampled = true
// 이후 모든 하위 서비스에 전파
traceparent: ...-01  (sampled flag)
+ 구현 간단, 오버헤드 낮음
- 에러/느린 요청을 놓칠 수 있음

Tail-based Sampling (테일 기반)

요청 완료 후 전체 트레이스를 보고 샘플링 결정. 에러나 지연이 큰 요청을 확실히 수집.
// 모든 스팬 수집 후 판단
collector.evaluate(trace):
  if trace.has_error: keep
  if trace.duration > 2s: keep
  if trace.status >= 500: keep
  else: probabilistic(0.1)
+ 중요한 트레이스 보장 수집
- 메모리/리소스 사용량 높음, 지연
