Incident Response Process

인시던트 대응 프로세스 | 감지부터 복구까지 5단계 체계적 대응

1. Incident Response 5 Phases 인시던트 대응 5단계
1
Detect
감지
- 모니터링 알림 수신
- 고객 신고 접수
- 자동화 감지 시스템
2
Classify
분류 (Severity)
- SEV1: 전체 서비스 장애
- SEV2: 주요 기능 장애
- SEV3: 부분 기능 장애
- SEV4: 경미한 이슈
3
Mitigate
완화
- 즉각적 영향 최소화
- Rollback / Feature Flag
- Traffic shift / Scale up
4
Recover
복구
- 근본 원인 해결
- 서비스 정상화 확인
- 모니터링 강화 감시
5
Review
사후 검토
- 포스트모템 작성
- 재발 방지 대책
- 프로세스 개선

3. Role Assignment 역할 할당

Incident Commander (IC)

전체 대응 조율 및 의사결정
타임라인 관리 및 기록
에스컬레이션 결정
외부 커뮤니케이션 승인

Communications Lead

내부 팀 상황 공유 (Slack)
Status Page 업데이트
고객 알림 발송
경영진 보고

Operations Lead

기술적 원인 분석
완화/복구 조치 실행
관련 엔지니어 소집
모니터링 및 검증
4. Status Page Update Flow 상태 페이지 업데이트 흐름
10:08 AM
Investigating - API 서비스 응답 지연이 감지되었습니다. 원인을 조사 중입니다.
10:15 AM
Identified - 데이터베이스 연결 풀 고갈이 원인으로 확인되었습니다. 긴급 조치를 진행합니다.
10:30 AM
Monitoring - DB 연결 풀 확장 및 재시작 완료. 서비스 복구를 모니터링 중입니다.
10:45 AM
Resolved - 모든 서비스가 정상 복구되었습니다. 포스트모템을 진행할 예정입니다.
