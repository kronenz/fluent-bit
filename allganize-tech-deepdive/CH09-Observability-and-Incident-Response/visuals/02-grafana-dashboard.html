4. Alert Rule Evaluation 알림 규칙 평가 흐름

Alert Rule 정의

- alert: HighErrorRate
  expr: |
    rate(http_errors[5m])
    / rate(http_total[5m])
    > 0.05
  for: 5m
  labels:
    severity: critical

Evaluation (평가)

evaluation_interval: 15s

pending → firing (5m 지속)
  → for 조건 충족 시 알림 발생

firing → resolved
  → 조건 미충족 시 해제
pending 상태에서 for 기간 동안 지속되면 firing

Notification (알림)

Contact Points:
  → Slack #alerts-critical
  → PagerDuty (on-call)
  → Email (team-leads)

Notification Policy:
  group_by: [alertname, cluster]
  group_wait: 30s
  repeat_interval: 4h
