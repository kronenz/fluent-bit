2. Alertmanager Logic 알림 관리 로직

Grouping (그룹핑)

유사한 알림을 하나의 알림으로 묶어서 알림 폭주(alert storm)를 방지합니다.
route:
  group_by: ['alertname','cluster']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
100개의 Pod 알림 → cluster별 1개 그룹 알림으로 축소

Inhibition (억제)

상위 알림이 발생하면 하위 알림을 자동 억제합니다. 노드 다운 시 Pod 알림 불필요.
inhibit_rules:
- source_match:
    alertname: 'NodeDown'
  target_match:
    alertname: 'PodNotReady'
  equal: ['node']
NodeDown 발생 → 해당 노드의 PodNotReady 알림 자동 억제

Silence (무음)

특정 알림을 일정 기간 무음 처리합니다. 유지보수 작업 시 사전 설정.
# Alertmanager UI에서 설정
Matchers:
  alertname="DeploymentScaling"
  namespace="staging"

Duration: 2h
Comment: "Scheduled scaling test"
Created by: ops-team
계획된 배포/스케일링 중 불필요한 알림 방지
3. Escalation Policy 에스컬레이션 정책 (단계적 확대)

L1 - On-Call

0 ~ 15분
On-Call Engineer
자동 PagerDuty 호출
Slack #alerts-p1 알림
Runbook 확인 및 초기 대응
15분 미응답

L2 - Team Lead

15 ~ 30분
팀 리드 / 시니어 엔지니어
추가 인력 투입 판단
고객 영향도 평가
임시 조치 (rollback 등)
30분 미해결

L3 - Management

30분+
Engineering Manager / VP
전사 인시던트 선언
고객 커뮤니케이션
외부 벤더 에스컬레이션
