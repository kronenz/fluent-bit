VNET & Subnet 설계

AKS CNI에서는 Pod마다 VNET IP가 필요합니다. /22 이상의 충분한 Subnet이 필요하며, Private Endpoint로 PaaS 서비스에 안전하게 접근합니다.

AKS + AAD Integration

Azure AD를 AKS 인증 백엔드로 사용합니다. AAD 그룹과 ClusterRoleBinding을 매핑하여 RBAC을 관리합니다. Conditional Access도 적용 가능합니다.

NSG (Network Security Group)

Subnet 레벨의 방화벽입니다. Inbound/Outbound 규칙으로 트래픽을 제어합니다. AKS Subnet에서는 필요한 포트만 허용합니다.

Azure Policy

조직의 거버넌스 규칙을 코드로 정의합니다. Deny(차단), Audit(감사), DeployIfNotExists(자동배포) 효과로 컴플라이언스를 강제합니다.
