Terraform 실행 흐름

Terraform Execution Flow - Write / Init / Plan / Apply 단계별 시뮬레이션

📝
Write .tf
코드 작성 / Configuration
➡
🔌
Init
프로바이더 초기화 / Providers
➡
📈
Plan
변경사항 미리보기 / Diff
➡
✅
Apply
인프라 반영 / Create/Update/Delete

🔗 리소스 의존성 그래프 (Resource Dependency DAG)

Write .tf 코드 작성

HCL(HashiCorp Configuration Language)로 인프라를 선언적으로 정의합니다. 리소스, 변수, 출력값을 코드로 표현합니다.

Init 초기화

terraform init은 프로바이더 플러그인 다운로드, 백엔드 설정, 모듈 설치를 수행합니다. .terraform/ 디렉토리에 저장됩니다.

Plan 실행 계획

현재 상태(state)와 코드(config)를 비교하여 변경사항을 미리 보여줍니다. + 생성, ~ 수정, - 삭제를 표시합니다.

Apply 적용

Plan의 결과를 실제 인프라에 반영합니다. DAG 순서대로 리소스를 생성/수정/삭제하고 State를 업데이트합니다.
