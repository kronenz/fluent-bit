Pod Lifecycle

Pod 전체 생명주기: 생성, 프로브, 종료, 재시작

KEY VISUAL
Step 0 / 0
kubectl apply Pending
Pod created in etcd. spec.nodeName empty. Waiting for scheduler.
Scheduling Pending
Scheduler assigns node. Binding created. kubelet picks up Pod spec.
Image Pull Pending
kubelet pulls container image based on imagePullPolicy (Always/IfNotPresent/Never).
Init Containers Pending
Init containers run sequentially. Each must complete successfully (exit 0) before next starts. init-db -> init-config -> ...
Main Containers Start Running
All main containers start simultaneously. postStart hooks execute (not guaranteed before ENTRYPOINT).
Startup Probe
If configured, blocks other probes until startup succeeds. For slow-starting apps. failureThreshold * periodSeconds = max startup time.
Liveness + Readiness Probes Running
Liveness: restart if fails. Readiness: remove from Service endpoints if fails. Both run periodically throughout container life.
Termination Terminating
preStop hook -> SIGTERM -> terminationGracePeriodSeconds (default 30s) -> SIGKILL. Removed from Service endpoints immediately.

Liveness Probe / 생존 프로브

"Is the container alive?"
Detects deadlocks, infinite loops, unresponsive state.
Methods: httpGet, tcpSocket, exec, grpc
Params: initialDelaySeconds, periodSeconds, failureThreshold
Failure Action: Container RESTART (kill + recreate)

Readiness Probe / 준비 프로브

"Is the container ready to receive traffic?"
App may be alive but loading cache, warming up, etc.
Methods: httpGet, tcpSocket, exec, grpc
Params: initialDelaySeconds, periodSeconds, successThreshold
Failure Action: Remove from Service ENDPOINTS (no traffic)

Startup Probe / 시작 프로브

"Has the application finished starting?"
For slow-starting apps (Java, ML models).
Blocks liveness/readiness until success.
Max time: failureThreshold x periodSeconds
While Running: Other probes DISABLED until success
1
Pod set to Terminating
API server sets deletionTimestamp. Pod removed from Service endpoints immediately. No new traffic routed.
▼
2
preStop Hook Executes
If defined, preStop hook runs (e.g., nginx -s quit, drain connections). Runs IN PARALLEL with endpoint removal.
Example: sleep 15 to allow in-flight requests to complete
▼
3
SIGTERM Sent
kubelet sends SIGTERM to PID 1 in the container. Application should catch this and begin graceful shutdown (close connections, flush buffers, save state).
▼
4
Grace Period Countdown
terminationGracePeriodSeconds (default: 30s). Application has this much time to shut down gracefully.
▼
5
SIGKILL (Force Kill)
If container still running after grace period, SIGKILL is sent. Process is forcefully terminated. Cannot be caught or ignored.
terminationGracePeriodSeconds: 30s countdown
