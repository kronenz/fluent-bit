CPU Throttling Mechanism / CPU 쓰로틀링 메커니즘

CPU is a compressible resource. When a container hits its CPU limit, the kernel CFS (Completely Fair Scheduler) throttles it by limiting CPU time slices.

CFS Period: 100ms (default)
CPU Limit 500m: Container gets 50ms of CPU time per 100ms period
CPU Limit 1000m: Container gets 100ms (full core) per period

Throttled containers are NOT killed - they just run slower. Check container_cpu_cfs_throttled_periods_total metric.

CPU는 압축 가능한(compressible) 리소스입니다. 컨테이너가 CPU 제한에 도달하면 커널 CFS가 CPU 시간 슬라이스를 제한하여 쓰로틀링합니다. 쓰로틀링된 컨테이너는 죽지 않고 느려집니다.

OOM Kill Mechanism / OOM Kill 메커니즘

Memory is an incompressible resource. Unlike CPU, memory cannot be throttled - when a container exceeds its memory limit, the kernel OOM killer terminates it.

Container OOM: Container exceeds its own memory limit -> killed by cgroup OOM
Node OOM: Node runs out of memory -> kubelet evicts pods by QoS priority

Container exit code: 137 (128 + SIGKILL=9). Reason: OOMKilled.
kubectl describe pod shows: Last State: Terminated, Reason: OOMKilled, Exit Code: 137

메모리는 비압축(incompressible) 리소스입니다. CPU와 달리 쓰로틀링이 불가능하여, 컨테이너가 메모리 제한을 초과하면 커널 OOM killer가 종료시킵니다. Exit code: 137.
