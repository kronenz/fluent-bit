3. cosign Signature Flow cosign 서명 및 검증 흐름
1
키 생성 (Key Generation)
cosign generate-key-pair
cosign.key (개인키) + cosign.pub (공개키) 생성 / KMS 연동 가능 (AWS KMS, GCP KMS, Vault)
2
이미지 서명 (Image Signing)
cosign sign --key cosign.key registry.io/myapp:v1.0
이미지 digest에 대해 서명 생성, 서명은 OCI artifact로 레지스트리에 함께 저장됨
3
서명 검증 (Signature Verification)
cosign verify --key cosign.pub registry.io/myapp:v1.0
배포 전 서명 유효성 확인. 검증 실패 시 배포 차단 (Admission Controller 연동)
4
Keyless Signing (Sigstore / Fulcio)
cosign sign --identity-token=$(gcloud auth print-identity-token) registry.io/myapp:v1.0
OIDC identity 기반 서명. 개인키 관리 불필요. GitHub Actions OIDC와 자연스럽게 연동

Admission Controller Policy (Kyverno 예시)
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: verify-image-signature
spec:
  rules:
  - name: check-cosign
    match:
      any:
      - resources:
          kinds: [Pod]
    verifyImages:
    - imageReferences: ["registry.io/myapp:*"]
      attestors:
      - entries:
        - keys:
            publicKeys: |
              -----BEGIN PUBLIC KEY-----
              MFkwEwYHKoZI...
              -----END PUBLIC KEY-----
