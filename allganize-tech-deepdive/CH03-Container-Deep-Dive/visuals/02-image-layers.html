1. Image Layer Stacking 이미지 레이어 스태킹
Dockerfile 명령어 → 레이어 생성
R/W Layer (컨테이너 실행 시 생성)~0 MB
ENTRYPOINT ["node","app.js"]0 MB
COPY ./app /app~15 MB
RUN apt-get install -y nodejs~180 MB
FROM ubuntu:22.04 (Base Image)~77 MB
각 레이어 설명:
■ Base Layer: ubuntu:22.04 기본 이미지
  모든 시스템 파일 포함 (읽기 전용)
■ Install Layer: 패키지 설치 레이어
  apt cache + 바이너리 포함 (가장 큰 레이어)
■ App Layer: 애플리케이션 코드 복사
  코드 변경 시 이 레이어만 재빌드
■ Config Layer: 메타데이터만 (크기 0)
  ENTRYPOINT, CMD, ENV, EXPOSE 등
■ R/W Layer: 컨테이너 실행 시 생성
  Copy-on-Write로 변경사항 저장
2. Union Filesystem Merge View Union FS 병합 뷰

Individual Layers (개별 레이어)

Layer 0 (base):
/bin/ bash, ls, cat...
/usr/ lib/, share/
/etc/ apt/, passwd

Layer 1 (apt install):
/usr/bin/ node
/usr/lib/ nodejs/
/var/cache/ apt/archives/

Layer 2 (copy app):
/app/ app.js, package.json
/app/ node_modules/

Layer 3 (entrypoint):
메타데이터만 (파일 없음)

Merged View (병합된 뷰)

/
├── bin/ bash, ls, cat...
├── usr/
│   ├── bin/ node
│   ├── lib/ ... nodejs/
│   └── share/
├── etc/ apt/, passwd
├── app/
│   ├── app.js
│   ├── package.json
│   └── node_modules/
├── var/
│   └── cache/ apt/
└── tmp/

OverlayFS: upperdir + lowerdir
모든 레이어가 하나의 파일시스템처럼 보임
3. Multi-stage Build 멀티스테이지 빌드

Build Stage (빌드 스테이지)

FROM golang:1.21 AS builder
WORKDIR /src
COPY . .
RUN go build -o /app/server

# 포함: Go toolchain, source code,
# build cache, dependencies...
1.2 GB
➔
COPY --from=builder
/app/server만 복사

Runtime Stage (런타임 스테이지)

FROM gcr.io/distroless/static
COPY --from=builder /app/server /server
ENTRYPOINT ["/server"]

# 포함: static binary만
# No shell, no package manager
# 최소 공격 표면
15 MB

최적화 포인트:
1. 멀티스테이지 빌드로 빌드 도구 제거
2. distroless / Alpine 베이스 이미지 사용
3. .dockerignore로 불필요한 파일 제외
4. 레이어 병합 (RUN 명령어 결합)
5. apt cache 삭제 (rm -rf /var/lib/apt/lists/*)
