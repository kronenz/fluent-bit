Filesystem & I/O Stack

I/O 스택 계층, OverlayFS, Copy-on-Write, I/O 스케줄러 비교

Step 1. I/O 스택 계층 (Application → Device)

Application (애플리케이션)
write(), read() 시스템 콜 호출
⬇
VFS (Virtual File System)
파일시스템 추상화 계층 - 통일된 인터페이스 제공
⬇
Filesystem (ext4, XFS, btrfs)
실제 파일시스템 드라이버 - inode, block 관리
⬇
Block Layer (블록 계층)
I/O 스케줄러, bio 요청 병합/정렬
⬇
Device Driver → Hardware
NVMe, SCSI, SATA 드라이버 → SSD/HDD

Step 2. OverlayFS 계층 구조 (Container Storage)

merged/ (통합 뷰 - 컨테이너가 보는 것)
/etc/nginx.conf /var/log/access.log /tmp/cache.dat /usr/bin/nginx
⬆ merge ⬆
upperdir/ (쓰기 가능 계층)
/var/log/access.log /tmp/cache.dat .wh.old-config
lowerdir/ (읽기 전용 - 이미지 레이어)
/etc/nginx.conf /usr/bin/nginx /lib/libc.so.6
workdir/ (OverlayFS 내부 작업 디렉토리)

Step 3. Copy-on-Write 동작 (컨테이너 쓰기 시)

lowerdir (읽기 전용)
/etc/nginx.conf (원본)
➔ COPY ➔
upperdir (쓰기 가능)
/etc/nginx.conf (복사본)

none (noop)

스케줄링 없이 FIFO 순서로 전달
NVMe SSD에 최적 (HW 큐 활용)
최저 지연NVMe 기본값
