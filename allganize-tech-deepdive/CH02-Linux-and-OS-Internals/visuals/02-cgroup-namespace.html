cgroup & Namespace 격리

컨테이너 격리의 핵심 기술: Namespace로 뷰 분리, cgroup으로 자원 제한

Step 1-2. Namespace 격리 계층 (Host vs Container View)

💻 Host (호스트 전체 보기)
PID NSPID 1,2,...30000+
NET NSeth0, docker0, veth*
MNT NS/, /var, /home, /data
UTS NShostname: prod-server-01
IPC NSAll shared memory, semaphores
USER NSuid: 0(root), 1000(app)
VS
📦 Container (격리된 뷰)
PID NSPID 1 (nginx) 만 보임
NET NSeth0 (veth pair)
MNT NS/app 만 접근 가능
UTS NShostname: my-container
IPC NS격리된 IPC 객체
USER NSuid: 0 → host uid 10000

Step 3-4. cgroup 자원 제한 (CPU Throttling & OOM Kill)

⚡ CPU 제한 (cpu.max)

0%
제한: 200ms / 100ms period (2 cores)

💾 Memory 제한 (memory.max)

0 MB
제한: 512 MB

Step 5-6. cgroup v1 vs v2 비교 (Hierarchy Structure)

cgroup v1 (리소스별 계층)

/sys/fs/cgroup/
  • cpu/
    • docker/container-abc
  • memory/
    • docker/container-abc
  • blkio/
    • docker/container-abc
⚠ 리소스마다 별도 트리 → 관리 복잡

cgroup v2 (통합 트리)

/sys/fs/cgroup/
  • system.slice/
    • docker-abc.scope
      • cpu.max, memory.max, io.max
      • 모든 리소스 통합 관리
  • user.slice/
✔ 단일 트리 → 간결하고 일관된 관리
