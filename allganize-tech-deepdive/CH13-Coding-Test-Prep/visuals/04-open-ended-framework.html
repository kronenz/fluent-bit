STAR-T 답변 프레임워크 STAR-T Framework

S

Situation 상황

프로젝트 배경과 맥락을 설명합니다. 어떤 환경에서, 어떤 팀에서 일했는지 구체적으로 기술합니다.
Describe the project context and environment.

➔
T

Task 과제

해결해야 할 구체적인 문제나 목표를 설명합니다. 기술적 제약사항도 포함합니다.
Define the specific problem or goal you needed to achieve.

➔
A

Action 행동

본인이 취한 구체적 기술적 행동을 설명합니다. 도구, 기술, 의사결정 과정을 포함합니다.
Explain your specific technical actions and decisions.

➔
R

Result 결과

정량적 성과와 영향을 제시합니다. 수치로 표현할 수 있으면 더 좋습니다.
Present quantifiable outcomes and impact metrics.

➔
T

Tie-back 연결

경험을 지원하는 포지션과 연결합니다. 이 경험이 왜 해당 역할에 적합한지 설명합니다.
Connect your experience to the role you are applying for.

BUILD 답변 구조 빌더 Answer Structure Builder

1

🎯 문제 정의 Problem Statement

해결하려는 문제를 명확하게 정의합니다. 기술적 맥락과 비즈니스 영향을 포함합니다.

Clearly define the problem with technical context and business impact.

예시: "API 응답시간이 P99 기준 3초를 초과하여 사용자 이탈률이 15% 증가했습니다"
Example: "P99 API latency exceeded 3s, causing 15% user drop-off increase"
2

💡 접근 방법 Approach

문제 분석 과정과 선택한 솔루션의 근거를 설명합니다. 대안도 함께 언급합니다.

Explain your analysis process and justification. Mention alternatives considered.

예시: "프로파일링으로 DB 쿼리 N+1 문제 식별 -> Redis 캐싱 vs 쿼리 최적화 비교 후 캐싱 선택"
Example: "Profiled DB queries, found N+1 issue -> compared Redis caching vs query optimization"
3

🔧 구현 Implementation

구체적인 기술 스택, 코드 구조, 배포 전략을 설명합니다. 핵심 코드 예시를 포함하면 좋습니다.

Describe technical stack, code structure, deployment strategy with code examples.

예시: "Redis Cluster 3노드 구성, @cache_result 데코레이터 구현, TTL 전략 수립, 카나리 배포로 점진적 적용"
Example: "3-node Redis Cluster, @cache_result decorator, TTL strategy, canary rollout"
4

🏆 성과 Outcome

정량적 결과와 배운 점을 제시합니다. Before/After 비교로 임팩트를 명확히 합니다.

Present quantifiable results with before/after comparison and lessons learned.

예시: "P99 레이턴시 3.2s -> 180ms (94% 개선), 사용자 이탈률 15% -> 3%, DB 부하 60% 감소"
Example: "P99 latency 3.2s -> 180ms (94% improvement), drop-off 15% -> 3%, DB load -60%"
