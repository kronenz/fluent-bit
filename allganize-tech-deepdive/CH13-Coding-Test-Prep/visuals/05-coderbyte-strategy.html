TIPS 핵심 팁 Key Tips

📖 문제 먼저 전부 읽기

10분 투자하여 모든 문제를 읽고 난이도를 파악합니다. 쉬운 문제부터 풀어서 자신감을 확보하세요.
Read all problems first (10 min). Start with easy ones for confidence.

✍ 의사코드 먼저 작성

바로 코딩하지 말고 주석으로 로직을 먼저 작성합니다. 구조가 잡히면 코드 변환이 쉬워집니다.
Write pseudocode in comments first. Structure makes coding easier.

🛠 엣지 케이스 체크리스트

빈 입력, 단일 요소, 최대값, 음수, 중복값을 항상 체크합니다. 테스트 케이스 통과의 핵심입니다.
Always check: empty input, single element, max value, negatives, duplicates.

🕑 5분 룰 적용

5분 안에 접근법이 안 떠오르면 과감히 건너뛰세요. 쉬운 문제를 확실히 풀고 돌아오는 것이 유리합니다.
No approach in 5 min? Skip it. Secure easy points first.

📝 Open-Ended 는 구조가 점수

정답이 없는 문제는 논리적 구조와 구체적 예시가 핵심입니다. STAR-T 프레임워크를 활용하세요.
For open-ended: logical structure + specific examples = high score.

🔎 마지막 10분 리뷰

변수명 개선, 불필요한 코드 제거, 주석 추가, print/debug 문 삭제를 최종 점검합니다.
Final 10 min: improve naming, remove debug code, add comments.
