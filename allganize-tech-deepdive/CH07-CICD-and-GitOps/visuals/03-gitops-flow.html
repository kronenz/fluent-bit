GitOps 핵심 원칙 Core Principles

선언적 Declarative

시스템의 원하는 상태를 선언적으로 정의. 명령형(kubectl run) 대신 선언형(YAML manifests) 사용. Define desired state declaratively, not imperatively.

버전 관리 Versioned

Git이 Single Source of Truth. 모든 변경은 Git 커밋으로 추적, 감사, 롤백 가능. Git as the single source of truth for all changes.

자동 적용 Automated

승인된 변경은 자동으로 시스템에 적용. 수동 개입 없이 지속적 조정(Continuous Reconciliation). Approved changes are automatically applied.
