PATCH Strategic Merge Patch vs JSON Patch

Strategic Merge Patch

Kubernetes 리소스 구조를 이해하고 지능적으로 병합. 리스트는 전략적으로 처리 (merge key 기반).
Kubernetes-aware intelligent merge. Lists handled by merge keys.
# strategic-merge-patch.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 5 # 변경
  template:
    spec:
      containers:
      - name: my-app
        resources:
          limits:
            memory: "1Gi"
✓ 장점 Pros
- YAML 형태로 직관적, 읽기 쉬움
- 부분 수정만 명시하면 나머지 보존
- Kubernetes 기본 패치 전략

JSON Patch (RFC 6902)

명시적 연산(add/remove/replace)으로 정밀 수정. 배열 인덱스 접근 가능.
Explicit operations for precise modifications. Array index access supported.
# json-patch.yaml
- op: replace
  path: /spec/replicas
  value: 5
- op: add
  path: /spec/template/spec/containers/0/resources/limits
  value:
    memory: "1Gi"
- op: remove
  path: /spec/template/metadata/annotations/debug
✓ 장점 Pros
- add/remove/replace 등 명시적 연산
- 배열 인덱스로 정밀 제어 가능
- 필드 삭제가 가능 (SMP는 불가)

Kustomize 핵심 개념 Key Concepts

Template-Free

Helm과 달리 템플릿 언어 없이 순수 YAML 패칭. kubectl에 내장되어 별도 설치 불필요. Pure YAML patching without template language, built into kubectl.

Overlay Pattern

Base에 공통 리소스, Overlay에 환경별 차이만 정의. DRY 원칙 준수로 중복 제거. Define common resources in base, differences in overlays.

Transformers

namePrefix, nameSuffix, commonLabels, commonAnnotations 등 전역 변환기로 일괄 수정. Global transformers for bulk modifications.
