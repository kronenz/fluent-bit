COMPARE CI 도구 비교 CI Tool Comparison

GitHub Actions

  • YAML 기반 워크플로우 (.github/workflows/)
  • GitHub 호스팅 러너 또는 셀프호스팅 러너
  • 이벤트 드리븐: push, PR, schedule, workflow_dispatch
  • Matrix strategy로 병렬 테스트 지원
  • Marketplace 액션 재사용 (actions/checkout 등)
  • 무료 티어: 퍼블릭 리포 무제한, 프라이빗 2,000분/월

Jenkins

  • Jenkinsfile (Declarative/Scripted Pipeline)
  • Master-Agent 아키텍처, 셀프호스팅 필수
  • 플러그인 생태계 (1,800+ 플러그인)
  • Parallel stages로 병렬 실행
  • Shared Library로 파이프라인 코드 재사용
  • 완전한 커스터마이징, 운영 부담 높음

GitLab CI

  • .gitlab-ci.yml 단일 파일 설정
  • GitLab Runner (Docker/K8s/Shell executor)
  • 내장 컨테이너 레지스트리 + CI/CD 통합
  • DAG (Directed Acyclic Graph) 파이프라인
  • include/extends로 템플릿 재사용
  • Auto DevOps: 자동 빌드/테스트/배포

핵심 개념 Key Concepts

CI (Continuous Integration)

개발자가 코드를 자주 메인 브랜치에 통합하고, 자동화된 빌드와 테스트로 품질을 검증하는 개발 방법론. Developers frequently merge code into main, with automated builds and tests to verify quality.

Pipeline as Code

CI/CD 파이프라인을 코드로 정의하여 버전 관리, 코드 리뷰, 재사용이 가능. YAML 파일로 파이프라인을 선언적으로 정의. Define CI/CD pipelines as code for version control and reuse.

Matrix Build

여러 OS, 언어 버전 조합을 자동으로 병렬 실행하여 호환성을 검증. Automatically run parallel combinations of OS and language versions for compatibility testing.
