Path-based & Host-based Routing (경로 및 호스트 기반 라우팅)

Host-based Routing (호스트 기반)
api.example.com→api-service:80
web.example.com→web-service:80
admin.example.com→admin-service:80
Path-based Routing (경로 기반)
app.example.com/api/*→api-service:80
app.example.com/web/*→web-service:80
app.example.com/static/*→cdn-service:80
app.example.com/api/users
→
Rule Match: /api/*
→
api-service:80
1Ingress는 Host 헤더로 먼저 가상 호스트 매칭
Ingress matches virtual host by Host header first
2매칭된 호스트 내에서 Path prefix/exact 매칭 수행
Within matched host, performs path prefix/exact matching
3가장 구체적인(longest prefix) 규칙이 우선 적용
Most specific (longest prefix) rule takes priority

클러스터 내부는 평문
1클라이언트가 HTTPS로 암호화된 요청 전송
Client sends TLS-encrypted request
2Ingress Controller가 K8s Secret(tls-cert)의 인증서로 TLS 복호화
Ingress decrypts using TLS cert from K8s Secret
3복호화된 HTTP 요청을 백엔드 서비스로 평문 전달
Forwards decrypted HTTP to backend (plaintext inside cluster)
4선택적으로 Backend TLS (re-encrypt) 설정 가능
Optionally can re-encrypt to backend (backend TLS)

Gateway API Hierarchy (게이트웨이 API 계층 구조)

GatewayClass
인프라 공급자 정의
Infrastructure provider (e.g., istio, cilium)
Gateway
리스너, 포트, TLS 설정
Listeners, ports, TLS config (cluster ops manage)
HTTPRoute
라우팅 규칙 (앱 팀 관리)
Routing rules (app team manages)
GRPCRoute
gRPC 전용 라우팅
gRPC-specific routing
TCPRoute
L4 TCP 라우팅
L4 TCP routing
Backend Services / Pods
최종 워크로드
1GatewayClass: 인프라 레벨 정의 (Istio, Cilium, Contour 등)
GatewayClass: infra-level definition by platform team
2Gateway: 클러스터 운영팀이 리스너/포트/TLS 설정 관리
Gateway: cluster ops manage listeners, ports, TLS
3HTTPRoute/GRPCRoute/TCPRoute: 앱 팀이 라우팅 규칙 독립 관리
Routes: app teams independently manage routing rules
4역할 분리 (RBAC): Ingress보다 세밀한 권한 위임 가능
Role separation: finer-grained delegation than Ingress
