1podSelector: {} → 네임스페이스의 모든 Pod에 정책 적용
Empty selector targets ALL pods in namespace
2Ingress + Egress 모두 비어있음 → 모든 인/아웃바운드 차단

Foundation for Zero Trust: only explicitly allowed traffic passes

# allow-frontend-to-api.yaml spec: podSelector: matchLabels: app: api # 대상: api Pod policyTypes: [Ingress] ingress: - from: - podSelector: matchLabels: app: frontend # 출발지: frontend Pod만 - namespaceSelector: matchLabels: env: production # 네임스페이스: production만 ports: - port: 8080 protocol: TCP

1podSelector로 정책 대상(api Pod) 지정
podSelector targets the api pods
2ingress.from에서 출발지 Pod 라벨 + 네임스페이스 라벨로 필터링
ingress.from filters by pod labels + namespace labels
3ports로 허용할 포트와 프로토콜 명시 (8080/TCP만 허용)
ports specifies allowed port + protocol (only 8080/TCP)

Egress Allow Rules (이그레스 허용 규칙)

# allow-api-egress.yaml spec: podSelector: matchLabels: app: api policyTypes: [Egress] egress: - to: - podSelector: matchLabels: app: db ports: - port: 5432 - to: - ipBlock: cidr: 0.0.0.0/0 except: - 10.0.0.0/8 # 내부 네트워크 제외 ports: - port: 443 protocol: TCP
ns: production
api
app: api
db
app: db
api→db:5432ALLOWED
api→external:443 (HTTPS)ALLOWED
api→10.0.5.10:443DENIED (internal)
api→external:80 (HTTP)DENIED (port)
1Egress 규칙 1: api → db Pod의 5432 포트만 허용
Egress rule 1: api to db pods on port 5432 only
2Egress 규칙 2: ipBlock으로 외부 CIDR의 443 포트 허용 (10.0.0.0/8 제외)
Egress rule 2: allow external CIDR on port 443, except 10.0.0.0/8
3내부 IP 대역(10.x)이나 허용되지 않은 포트(80)로의 요청은 차단
Internal IPs or non-allowed ports are blocked

Before / After Policy Application (정책 적용 전후 비교)

BEFORE (정책 적용 전)

No NetworkPolicy

모든 Pod 간 통신 허용 (default allow all)

any pod→any podOPEN
external→any podOPEN
attacker→db:5432OPEN
compromised→lateral moveOPEN
Risk: 침해된 Pod에서 클러스터 전체로 lateral movement 가능
Compromised pod can reach any service in the cluster
AFTER (정책 적용 후)

With NetworkPolicy

최소 권한 원칙 적용 (Zero Trust)

frontend→api:8080ALLOWED
api→db:5432ALLOWED
attacker→db:5432BLOCKED
compromised→lateral moveBLOCKED
Benefit: 침해 범위가 정책으로 격리, blast radius 최소화
Blast radius minimized by policy isolation
1정책 없음: K8s는 기본적으로 모든 Pod 간 통신을 허용 (flat network)
No policy: K8s default allows all pod-to-pod communication
2Default Deny 적용 후 필요한 통신만 명시적으로 허용
Apply default-deny first, then explicitly allow needed flows
3Lateral movement 차단: 침해된 Pod가 다른 서비스에 접근 불가
Lateral movement blocked: compromised pod cannot reach other services
