1XDP: NIC 드라이버 레벨에서 최초 패킷 필터링 (DDoS 방어)
XDP: earliest packet filtering at NIC driver level
2TC: 네트워크 스택 진입 시 L3/L4 처리 (라우팅, NAT, LB)
TC: L3/L4 processing at network stack entry
3Policy: Identity 기반 네트워크 정책 평가 및 적용
Policy: identity-based network policy evaluation
4Socket: 최종 목적지 소켓으로 전달 (또는 TC egress로 포워딩)
Socket: delivered to destination (or forwarded via TC egress)

kube-proxy vs Cilium eBPF (kube-proxy 대체)

Traditional kube-proxy

1Pod sends to ClusterIP
2Packet enters netfilter
3iptables chain traversal
4Random probability rules
5DNAT to pod IP
6Conntrack entry created
7Packet forwarded
Rules: O(n) per service
Updates: full iptables rewrite
10,000 services = ~50,000 rules
VS

Cilium eBPF replacement

1Pod sends to ClusterIP
2eBPF socket hook intercepts
3BPF map lookup (O(1) hash)
4Direct DNAT to pod IP
5Shortcut to destination
Lookup: O(1) hash map
Updates: single map entry
10,000 services = same speed
1kube-proxy: iptables 규칙 수가 서비스 수에 비례 → O(n) 성능 저하
kube-proxy: iptables rules scale linearly with services
2Cilium: BPF hash map으로 O(1) 조회 → 서비스 수와 무관한 성능
Cilium: BPF hash map O(1) lookup, constant performance
3Cilium은 소켓 레벨 훅으로 불필요한 네트워크 스택 순회 회피
Cilium uses socket-level hooks to bypass unnecessary stack traversal
