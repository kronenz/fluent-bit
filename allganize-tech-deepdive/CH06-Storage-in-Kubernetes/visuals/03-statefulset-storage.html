Ordered Deployment & Deletion: 순서 보장 배포/삭제
▶ 배포 순서 (Deploy Order)
1. mongodb-0 생성
Ready 대기 → ✓
2. mongodb-1 생성
0번이 Ready 후 시작 → ✓
3. mongodb-2 생성
1번이 Ready 후 시작 → ✓
■ 삭제 순서 (Delete Order) — 역순
1. mongodb-2 삭제
가장 마지막 Pod 먼저
2. mongodb-1 삭제
2번 삭제 완료 후
3. mongodb-0 삭제
1번 삭제 완료 후
ⓘ 참고: PVC는 StatefulSet 삭제 시에도 보존됨 (데이터 보호).
수동 삭제 필요. podManagementPolicy: Parallel로 병렬 배포 가능.
