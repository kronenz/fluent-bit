# 08. 면접 당일 최종 체크리스트

> **TL;DR**
> - 면접 성패의 30%는 **준비 상태**에서 결정된다. 기술 실력이 아니라 환경과 마인드셋.
> - D-3부터 시작하는 체계적 준비로 당일 불안을 최소화한다.
> - 예상치 못한 상황(네트워크 끊김, 질문 모름 등)에 대한 **비상 시나리오**까지 준비한다.

> **면접 빈출도**: ★★★
> **예상 소요 시간**: 10min (체크리스트 확인)

---

## D-3: 사전 준비

### 정보 확인

- [ ] 면접 일시, 장소 (또는 화상 면접 링크) 재확인
- [ ] 면접관 이름, 직함 확인 (LinkedIn 프로필 간단히 확인)
- [ ] 면접 형식 확인 (기술 면접 / 컬쳐핏 / 코딩 테스트 / 복합)
- [ ] 예상 소요 시간 확인
- [ ] Allganize 최근 뉴스, 블로그 포스트 확인 (면접 시 활용)

### 콘텐츠 준비

- [ ] CH14 전체 문서 1회 통독
- [ ] 자기소개 1분/3분 소리 내어 연습 (최소 3회)
- [ ] 지원동기 3버전 키워드 암기
- [ ] 장단점 스크립트 연습
- [ ] 역질문 5개 선정 + 순서 결정
- [ ] 기술 면접 핵심 토픽 복습 (CH01-CH13 TL;DR 훑기)

---

## D-1: 환경 점검

### 화상 면접인 경우

- [ ] **카메라** 테스트 — 얼굴이 정중앙, 눈높이에 위치
- [ ] **마이크** 테스트 — 음질 확인, 유선 이어폰 준비 (블루투스는 끊김 위험)
- [ ] **네트워크** 속도 테스트 — 최소 10Mbps 이상 확인
- [ ] **백업 네트워크** — 모바일 핫스팟 준비 + 데이터 잔량 확인
- [ ] **화상회의 앱** — Zoom/Google Meet/Teams 최신 버전 업데이트
- [ ] **배경** — 깔끔한 벽면 또는 블러 배경 설정
- [ ] **조명** — 얼굴에 그림자 없도록. 모니터 앞 조명 또는 창가 자연광
- [ ] **방해 요소** 제거 — 알림 끄기, 반려동물/가족에게 미리 알리기, 문 잠금

```
환경 테스트 순서:
1. Zoom 등 앱 실행
2. 설정 > 오디오 > 마이크/스피커 테스트
3. 설정 > 비디오 > 카메라 확인
4. 친구/가족에게 테스트 콜 요청 (실제 통화 품질 확인)
5. 백업: 스마트폰에도 같은 앱 설치 + 로그인 확인
```

### 대면 면접인 경우

- [ ] **경로 확인** — 네이버/카카오 지도로 소요 시간 확인
- [ ] **도착 시간** — 면접 15분 전 건물 도착 목표 (10분 전 로비)
- [ ] **복장** — 비즈니스 캐주얼 (깔끔한 셔츠 + 면바지/치노). IT 스타트업이므로 정장은 과함
- [ ] **준비물** — 신분증, 이력서 프린트 2부, 노트북(코딩 테스트 대비), 펜/메모장
- [ ] **주차** — 자차 시 주차 가능 여부 확인

## D-Day: 면접 당일

### 면접 2시간 전

- [ ] 가벼운 식사 (공복도, 과식도 피하기)
- [ ] 커피 1잔 (각성 효과, 단 과다 섭취 시 떨림 주의)
- [ ] CH14 핵심 키워드 최종 점검 (스크립트 통독 X, 키워드만)
- [ ] 자기소개 1회 소리 내어 연습 (워밍업)

### 면접 30분 전

- [ ] 화장실 다녀오기
- [ ] 스마트폰 무음 모드 (진동도 끄기)
- [ ] 물 한 잔 준비 (목 마를 때 대비)
- [ ] 화상 면접: PC 재부팅 → 불필요한 앱 종료 → 미팅 룸 접속 대기
- [ ] 대면 면접: 건물 로비 도착 → 안내에 따라 이동

### 면접 5분 전

- [ ] 심호흡 3회 (4초 들숨 - 7초 날숨)
- [ ] 어깨 힘 빼기 — 의식적으로 어깨를 내린다
- [ ] 미소 연습 — 거울 보며 자연스러운 미소 확인
- [ ] 마인드셋 전환: "심사받는 자리"가 아니라 **"서로 알아가는 대화"**

## 면접 중 행동 가이드

### 첫인상 (처음 30초)

```
1. 밝은 표정으로 인사: "안녕하세요, [이름]입니다. 만나뵙게 되어 반갑습니다."
2. 화상: 카메라를 보며 인사 (모니터가 아닌 카메라 렌즈)
3. 대면: 적절한 악수 (너무 세지도, 약하지도 않게)
4. 면접관이 말할 때 고개 끄덕이기 (경청의 신호)
```

### 답변 구조

```
모든 답변은 STAR 구조로:

S (Situation): "~한 상황이었습니다"
T (Task): "제가 해야 했던 것은 ~였습니다"
A (Action): "그래서 저는 ~했습니다"
R (Result): "결과적으로 ~가 되었습니다"

시간 배분: 전체 답변 1-2분 이내
너무 짧으면 깊이 없어 보이고, 너무 길면 횡설수설
```

### 모르는 질문이 나왔을 때

```
최악의 반응: "모르겠습니다" (침묵)
차악의 반응: 아는 척하며 틀린 답변

좋은 반응:
"솔직히 그 부분은 실무 경험이 없어서 정확히는 모르겠습니다.
 다만, [관련 경험]을 바탕으로 추측하면 ~일 것 같고,
 입사 후 빠르게 학습하겠습니다."

→ 솔직함 + 유추 능력 + 학습 의지를 동시에 보여준다
```

### 긴장 관리

```
손이 떨릴 때 → 테이블 위에 가볍게 올려놓기 (무릎 위에 숨기지 않기)
말이 빨라질 때 → 의식적으로 문장 끝에 1초 멈추기
머리가 하얘질 때 → "잠시 정리하겠습니다" 하고 5초 생각
목소리가 떨릴 때 → 물 한 모금 마시기 (자연스러운 휴식)
```

## 비상 시나리오

### 시나리오 1: 네트워크 끊김 (화상 면접)

```
즉시 대응:
1. 스마트폰 핫스팟 연결 (30초 이내)
2. 동일 링크로 재접속
3. "죄송합니다, 네트워크 이슈가 있었습니다" (간단히 사과)

사전 예방:
- 면접관 이메일 또는 채용 담당자 연락처 저장
- "혹시 접속이 끊기면 [전화번호]로 연락 부탁드립니다" 사전 공유
```

### 시나리오 2: 예상치 못한 질문

```
"5년 후 이 회사가 망하면 어떡할 건가요?"
"당신을 동물에 비유하면?"
"마지막으로 실패한 경험을 말해주세요"

대응 원칙:
1. 당황하지 않기 (이런 질문은 반응을 보는 것)
2. 3초 생각한 후 답변 (급하게 말하면 더 당황)
3. 정답은 없다. 사고 과정을 보여주는 게 핵심
```

### 시나리오 3: 압박 면접

```
"경력에 비해 DevOps 경험이 없네요"
"이 나이에 새로운 분야를 시작하는 게 가능할까요?"

대응 원칙:
1. 감정적으로 반응하지 않기 (이것이 테스트)
2. 인정할 건 인정하고, 피벗하기 (CH14-03 참고)
3. "좋은 지적이십니다" → 공격을 대화로 전환
4. 구체적 근거로 반박: "DevOps 타이틀은 짧지만,
   실제로 한 일의 본질은..."
```

### 시나리오 4: 면접 시간 초과

```
면접관이 시간을 넘겨 이야기할 때:
→ 좋은 신호. 관심이 있다는 의미. 자연스럽게 따라간다.

내가 답변이 길어질 때:
→ "좀 길어졌는데, 요약하면 ~입니다" 로 정리
```

## 면접 후 행동

### 직후 (30분 이내)

- [ ] 면접 내용 메모 (질문, 내 답변, 분위기, 면접관 반응)
- [ ] 잘 답한 것 / 아쉬운 것 기록
- [ ] 채용 담당자에게 감사 메시지 (간결하게)

```
감사 메시지 예시:

"오늘 면접 기회 주셔서 감사합니다.
Allganize의 기술적 도전과 팀 문화에 대해 들으면서
합류하고 싶은 마음이 더 커졌습니다.
좋은 결과 기대하겠습니다."
```

### 1주일 후

- [ ] 결과 연락이 없으면 채용 담당자에게 **한 번만** 팔로업
- [ ] 면접 복기 내용을 바탕으로 부족했던 부분 학습

## 마인드셋 정리

### 면접 전에 읽을 5문장

```
1. "나는 10년 넘게 인프라를 해온 전문가다. 자격이 있다."

2. "면접은 시험이 아니라 대화다. 서로 맞는지 확인하는 자리다."

3. "모르는 건 모른다고 말해도 된다.
   솔직함이 아는 척보다 낫다."

4. "이 회사가 나를 뽑지 않아도,
   내 가치가 줄어드는 것은 아니다."

5. "준비한 만큼 나온다. 나는 충분히 준비했다."
```

### 자신감 앵커링

```
면접 전 1분:

눈을 감고, 지금까지 가장 잘 해결했던 기술 문제를 떠올린다.
그때의 성취감, 동료들의 반응, 자부심을 느낀 그 순간.
그 감정을 3초간 유지한다.

눈을 뜬다. 이 사람이 면접을 보러 간다.
```
