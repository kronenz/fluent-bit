# 02. 왜 Allganize인가 — 지원동기 3가지 버전

> **TL;DR**
> - 지원동기는 **기술 성장 / 글로벌 비전 / 임팩트** 3가지 축으로 준비해 면접 분위기에 맞게 선택한다.
> - "왜 이 회사?"에 대한 답은 회사 정보를 얼마나 조사했는지가 곧 성의의 증거다.
> - 꼬리 질문(왜 지금? 다른 회사는? 이직 사유?)까지 대비해야 완성이다.

> **면접 빈출도**: ★★★
> **예상 소요 시간**: 20min

---

## Allganize 기업 분석 요약

면접에서 활용할 팩트를 정리해둔다. **숫자와 고유명사**가 있어야 "조사했다"는 인상을 준다.

| 항목 | 내용 |
|---|---|
| 핵심 제품 | **Alli** — 기업용 AI/LLM 서비스 (문서 이해, 챗봇, 자동화) |
| 기술 스택 | LLM 기반 NLP, Kubernetes, 클라우드 네이티브 |
| 시장 | 한국 + 일본 + 글로벌 (B2B SaaS) |
| 특이점 | **도쿄 증시 상장 추진** — 글로벌 스케일업 단계 |
| 팀 문화 | 스타트업 속도 + 엔터프라이즈 고객 대응 |
| DevOps 역할 | 서비스 인프라 안정성 + 배포 자동화 + 글로벌 확장 지원 |

---

## 버전 1: 기술 성장 중심

> 면접관이 **기술 리드 / CTO**일 때 적합

```
솔직하게 말씀드리면, 저는 지금까지 인프라의 "하드웨어 쪽"에 가까운 일을 해왔습니다.
서버 랙킹부터 OS 설정, 네트워크 구성까지요.

그런데 최근 2-3년간 AI 데이터센터 프로젝트를 하면서,
쿠버네티스와 클라우드 네이티브 기술이
인프라의 미래라는 걸 확실히 체감했습니다.

Allganize는 LLM 서비스를 실제 프로덕션으로 운영하는 회사잖아요.
GPU 스케줄링, 모델 서빙, 트래픽 관리 같은
AI 특화 인프라 챌린지가 매일 발생하는 환경이라고 생각합니다.

저는 그 환경에서 DevOps 엔지니어로 성장하고 싶습니다.
베어메탈부터 이해하는 제 배경이,
클라우드 위에서 발생하는 문제를 더 깊이 진단하는 데 도움이 될 거라 생각합니다.
```

**핵심 키워드**: AI 인프라 챌린지, 클라우드 네이티브 성장, 깊이 있는 진단

---

## 버전 2: 글로벌 비전 중심

> 면접관이 **경영진 / HR**일 때 적합

```
Allganize에 관심을 갖게 된 계기는 도쿄 증시 상장 소식이었습니다.

한국 AI 스타트업이 일본 시장을 교두보로 글로벌 확장을 한다는 건,
기술력에 대한 자신감이 있다는 뜻이라고 생각했습니다.

저는 커리어의 다음 단계에서
"한국에서만 통하는 엔지니어"가 아니라
"글로벌 서비스를 운영할 수 있는 엔지니어"가 되고 싶습니다.

멀티 리전 배포, 글로벌 트래픽 관리, 규정 준수 같은 과제들을
Allganize에서 실전으로 경험하고 싶습니다.

10년간 다양한 환경에서 인프라를 다뤄본 적응력이
새로운 시장, 새로운 기술 환경에서도 강점이 될 수 있다고 생각합니다.
```

**핵심 키워드**: 글로벌 확장, 도쿄 상장, 멀티 리전, 적응력

---

## 버전 3: 임팩트 중심

> **실무 팀 리드** 면접이나 캐주얼한 분위기일 때 적합

```
저는 지금까지 "인프라를 만들어주는 쪽"에서 일했습니다.
컨설팅이다 보니, 고객에게 설계를 넘기면 제 손을 떠나거든요.

이제는 직접 만들고, 직접 운영하고,
문제가 터지면 직접 고치는 환경에서 일하고 싶습니다.

Allganize의 Alli 서비스는 실제 기업 고객이 매일 쓰는 제품이잖아요.
제가 구축한 파이프라인이 서비스 안정성에 직접 영향을 미치고,
제가 개선한 배포 프로세스가 개발팀의 속도를 높이는 —
그런 체감 가능한 임팩트를 원합니다.

컨설팅에서 쌓은 다양한 환경 경험이
실제 서비스 운영에서 응용력으로 발휘될 거라 확신합니다.
```

**핵심 키워드**: 직접 운영, 체감 가능한 임팩트, 서비스 안정성

## 예상 꼬리 질문 + 답변

### Q1. "왜 하필 지금 이직하려고 하세요?"

```
현재 회사에서 AI 데이터센터 프로젝트를 하면서
클라우드 네이티브와 DevOps에 대한 확신이 생겼습니다.
컨설팅으로는 이 분야를 깊이 파고들기 어렵다고 판단했고,
직접 서비스를 운영하는 환경으로 옮기기로 결심했습니다.
Allganize가 마침 DevOps 엔지니어를 채용하고 있어서 타이밍이 맞았습니다.
```

### Q2. "다른 AI 회사도 많은데, 왜 Allganize?"

```
세 가지 이유입니다.
첫째, B2B AI SaaS라는 비즈니스 모델이 안정적이고 확장 가능합니다.
둘째, 도쿄 상장을 추진할 정도로 글로벌 지향적입니다.
셋째, 저와 같은 인프라 배경을 가진 사람이
LLM 서비스 운영이라는 새로운 도메인에서 성장할 여지가 큽니다.
단순히 유명한 회사가 아니라, 제 성장 방향과 맞는 회사를 찾았습니다.
```

### Q3. "현재 회사에 대한 불만이 있으신 건가요?"

```
불만이라기보다 성장 방향의 차이입니다.
현재 회사는 좋은 곳이고 많이 배웠지만,
컨설팅 특성상 하나의 서비스를 깊이 파고들기가 어렵습니다.
저는 이제 한 서비스의 인프라를 처음부터 끝까지 책임지고
장기적으로 개선해나가는 경험을 하고 싶습니다.
```

### Q4. "Allganize에서 뭘 하고 싶으세요? 구체적으로요."

```
우선 첫 6개월은 기존 인프라와 배포 파이프라인을 완전히 이해하는 데 집중하겠습니다.
그 이후에는 세 가지를 하고 싶습니다.

1. CI/CD 파이프라인 최적화 — 배포 속도와 안정성 개선
2. 옵저버빌리티 강화 — LLM 서비스 특성에 맞는 모니터링 구축
3. 글로벌 확장 대비 — 멀티 리전 배포 자동화

물론 팀의 우선순위에 따라 유연하게 조정할 준비가 되어 있습니다.
```

### Q5. "Allganize 제품 써보셨어요?"

```
네, Alli 데모를 직접 사용해봤습니다.
문서를 업로드하면 AI가 내용을 이해하고 질문에 답하는 기능이 인상적이었습니다.
엔지니어 관점에서는, 이 서비스가 안정적으로 돌아가려면
문서 파싱 → LLM 추론 → 응답 생성 파이프라인의
각 단계가 얼마나 복잡할지 상상이 되면서 흥미로웠습니다.
특히 대용량 문서 처리 시 스토리지 I/O와 모델 서빙 레이턴시 관리가
DevOps 관점에서 핵심 챌린지일 것 같습니다.
```
