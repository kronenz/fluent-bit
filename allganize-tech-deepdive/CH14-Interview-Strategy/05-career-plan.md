# 05. 커리어 플랜 — 1년 / 3년 / 5년 성장 로드맵

> **TL;DR**
> - 단기(1년): 클라우드 네이티브 DevOps **실전 역량 확보**. Allganize 인프라 완전 이해.
> - 중기(3년): **플랫폼 아키텍트** 수준. 인프라 설계를 주도할 수 있는 역량.
> - 장기(5년): **테크 리드**. 기술 방향 설정 + 팀 빌딩. Allganize 글로벌 확장의 인프라 리더.

> **면접 빈출도**: ★★☆
> **예상 소요 시간**: 15min

---

## 질문의 의도 파악

면접관이 커리어 플랜을 묻는 진짜 이유:

1. **이 사람이 오래 다닐 것인가?** → 회사 성장과 연결된 플랜이어야 함
2. **현실적인 자기 인식이 있는가?** → 너무 낮거나 너무 높으면 위험 신호
3. **자기 주도적인 사람인가?** → "시키는 대로 하겠습니다"는 최악의 답변

---

## 통합 스크립트

```
커리어 플랜을 세 단계로 말씀드리겠습니다.

[1년 — 실전 역량 확보]
첫 1년은 Allganize의 인프라를 완전히 이해하는 데 집중하겠습니다.
기존 아키텍처, 배포 파이프라인, 모니터링 체계를 파악하고,
팀의 온콜 로테이션에 참여하면서 실제 운영 감각을 쌓겠습니다.
동시에 Kubernetes와 Terraform 실무 역량을
프로덕션 레벨로 끌어올리는 게 1년 목표입니다.

[3년 — 플랫폼 아키텍트]
3년 차에는 인프라 설계를 주도할 수 있는 수준이 되고 싶습니다.
Allganize가 글로벌로 확장하면 멀티 리전 배포,
국가별 데이터 규정 준수, 트래픽 라우팅 같은 과제가 생길 텐데,
그 아키텍처를 설계하고 구현하는 핵심 멤버가 되고 싶습니다.
CKA, AWS SA Professional 같은 자격증도 이 시기에 취득하겠습니다.

[5년 — 테크 리드]
5년 후에는 DevOps/인프라 팀의 기술 방향을 설정하고,
주니어 엔지니어를 육성할 수 있는 테크 리드가 되고 싶습니다.
Allganize의 도쿄 상장 이후 글로벌 확장이 본격화되는 시점에서,
인프라 조직의 체계를 잡는 역할을 하고 싶습니다.
```

## 상세 로드맵: Allganize 성장과 연계

### 1년차: Cloud Native Practitioner

```
[Q1-Q2] 온보딩 + 기존 시스템 이해
├── Allganize 인프라 아키텍처 문서 정독 + 멘토와 페어 운영
├── 기존 CI/CD 파이프라인 전체 흐름 파악
├── 모니터링/알럿 체계 이해 (Prometheus, Grafana 등)
└── 첫 온콜 참여 — 실제 장애 대응 경험

[Q3-Q4] 기여 시작
├── 기존 파이프라인의 작은 개선점 PR 제출
├── 반복 수동 작업 자동화 1-2건
├── 내부 기술 문서 작성/개선
└── CKA 자격증 취득
```

**Allganize 연결**: 이 시기 Allganize는 상장 준비로 서비스 안정성이 최우선. 안정적 운영 기여가 가장 큰 가치.

### 3년차: Platform Architect

```
[2년차]
├── 주요 인프라 설계 논의에 참여
├── IaC 표준화 주도 (Terraform 모듈화)
├── 비용 최적화 프로젝트 리드
└── AWS SA Professional / GCP Professional 취득

[3년차]
├── 멀티 리전 배포 아키텍처 설계 참여
├── 글로벌 CDN/트래픽 관리 전략 수립
├── SRE 프랙티스 도입 (SLO/SLI/에러 버짓)
└── 기술 블로그 외부 발표 시작
```

**Allganize 연결**: 도쿄 상장 이후 일본/글로벌 고객 확대. 멀티 리전 인프라가 핵심 과제. 이 시기에 아키텍처 역량이 직접적 기여.

### 5년차: Tech Lead

```
[4년차]
├── 인프라 팀 기술 로드맵 작성 참여
├── 채용 면접 참여 + 주니어 멘토링
├── 크로스 팀 프로젝트 리드 (인프라 × 개발)
└── 장애 포스트모템 문화 정착

[5년차]
├── DevOps/인프라 팀 기술 방향 설정
├── 글로벌 인프라 조직 체계 수립
├── 기술 커뮤니티 발표/기고 활동
└── 차세대 인프라 전략 (엣지, 서버리스 등) 평가
```

**Allganize 연결**: 글로벌 본격 확장기. 여러 리전에 인프라 팀이 분산될 수 있고, 표준화와 가이드라인이 필요한 시점. 테크 리드의 역할이 조직 성장에 직결.

## 예상 꼬리 질문

### Q1. "5년 후에 이직 안 하시겠어요?"

```
제가 원하는 성장이 Allganize 안에서 가능하다면 이직할 이유가 없습니다.
오히려 한 조직에서 인프라를 처음부터 성장시키는 경험이
커리어에서 가장 값진 자산이 될 거라고 생각합니다.

5년이면 Allganize의 글로벌 인프라가 상당히 커져 있을 텐데,
그 성장을 함께한 사람으로서의 경험은
다른 어디서도 얻기 어려운 것이잖아요.
```

### Q2. "CTO가 되고 싶으신 건가요?"

```
지금 시점에서 CTO까지 말하기엔 솔직히 이릅니다.
우선 5년 안에 테크 리드로서 기술적 깊이와
팀 리딩 역량을 검증받는 게 현실적인 목표입니다.

그 이후의 경로는 그때의 상황과
회사의 필요에 따라 유연하게 생각하고 있습니다.
```

### Q3. "플랜대로 안 되면 어떡하세요?"

```
사실 플랜대로 되는 경우가 드문 것도 알고 있습니다.
중요한 건 방향성이지 정확한 타임라인은 아니라고 생각해요.

"클라우드 네이티브 역량 → 아키텍처 설계 → 기술 리더십"이라는
방향만 유지하면, 속도는 상황에 맞게 조절할 수 있습니다.
```

### Q4. "매니지먼트와 IC(Individual Contributor) 중 뭘 선호하세요?"

```
현재로서는 IC 트랙에 더 끌립니다.
기술적 깊이를 더 쌓아야 하는 단계라고 생각하거든요.

다만 3-5년 후에는 팀이 커지면서 자연스럽게
기술 리더십과 피플 매니지먼트를 병행하게 될 수 있다고 보고,
그 부분에 대한 준비도 필요하다고 생각합니다.
```
