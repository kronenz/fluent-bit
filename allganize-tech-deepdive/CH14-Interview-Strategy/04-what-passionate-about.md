# 04. "요즘 뭐에 빠져있어요?" — 열정을 보여주는 3가지 답변

> **TL;DR**
> - 이 질문의 본질은 **"이 사람이 자기 일을 좋아하는가?"**를 확인하는 것이다.
> - 문제 해결 / 자동화 / 임팩트 — 3가지 축으로 준비하되, 외운 티가 나면 역효과다.
> - 핵심은 **구체적인 에피소드 + 그때 느낀 감정**을 자연스럽게 전달하는 것.

> **면접 빈출도**: ★★☆
> **예상 소요 시간**: 15min

---

## 이 질문이 나오는 맥락

면접 중반~후반, 기술 질문이 끝나고 분위기가 풀릴 때 나온다.
변형 질문들:

- "요즘 관심 있는 기술이 뭐예요?"
- "퇴근 후에도 기술 공부하시는 편이에요?"
- "최근에 재미있게 본 기술 콘텐츠가 있나요?"
- "일할 때 가장 재미있는 순간이 언제예요?"

**함정**: 너무 진지하게 답하면 "면접용 답변"으로 느껴진다. 자연스러운 톤이 중요하다.

---

## 답변 1: 문제 해결 — "원인을 찾았을 때의 쾌감"

```
저는 시스템에서 이상한 현상이 발생하면
그 원인을 찾을 때까지 못 놓는 편이에요.

최근에 GPU 서버에서 추론 성능이 들쭉날쭉한 이슈가 있었는데,
처음엔 모델 문제인 줄 알았거든요.
근데 파고 들어가보니까 NUMA 노드 배치 문제였어요.
GPU가 연결된 PCIe 슬롯과 메모리가 다른 NUMA 노드에 있어서
메모리 접근할 때마다 지연이 생겼던 거죠.

그걸 찾아냈을 때의 쾌감이... 좀 중독성이 있어요. (웃음)
문제의 원인이 예상과 완전히 다른 레이어에 있을 때,
"아, 이래서 시스템을 넓게 봐야 하는구나" 하는 느낌이 들면서
이 일을 계속하고 싶다는 생각이 듭니다.
```

**왜 이 답변이 좋은가**: 구체적 에피소드 + 감정("쾌감", "중독성") + 직무 연결("시스템을 넓게 봐야")

---

## 답변 2: 자동화 — "반복 작업을 없애는 재미"

```
좀 웃길 수도 있는데, 저는 반복 작업을 발견하면 참을 수가 없어요.
"이거 스크립트로 만들면 되는데..." 하는 생각이 자동으로 들어요.

예전에 서버 초기 세팅을 수동으로 하던 팀에 있었는데,
OS 설치부터 패키지, 보안 설정, 사용자 계정까지
체크리스트가 A4 3장이었거든요.
그걸 Ansible 플레이북으로 만들어서 한 번에 돌리게 바꿨을 때,
팀원들이 "이게 된다고?" 하는 반응이 너무 좋았어요.

요즘은 Terraform이랑 GitHub Actions로
인프라 프로비저닝부터 배포까지 코드로 관리하는 걸 연습하고 있는데,
이게 또 재미있어요.
코드 한 줄로 인프라가 만들어지는 그 느낌이요.
```

**왜 이 답변이 좋은가**: 성격이 드러남("참을 수가 없어요") + 팀 임팩트 + 현재 학습 상태 자연스럽게 노출

---

## 답변 3: 임팩트 — "내가 만든 것이 돌아가는 걸 볼 때"

```
제가 이 일을 좋아하는 이유 중 하나가,
결과가 눈에 보인다는 거예요.

코드를 짜는 개발자분들은 결과가 UI나 기능으로 보이잖아요.
인프라는 좀 다르긴 한데, 저는 다른 방식으로 결과를 봐요.

모니터링 대시보드에서 에러율이 뚝 떨어지는 그래프,
배포 시간이 30분에서 5분으로 줄어든 파이프라인 로그,
서버 리소스 사용률이 최적화된 후의 깔끔한 메트릭 —
이런 걸 볼 때 "아, 일 잘했다" 하는 느낌이 들어요.

Allganize에서 Alli 서비스의 인프라를 관리하게 되면,
실제 고객이 사용하는 서비스의 안정성 지표를 직접 개선하는 거잖아요.
그게 정말 기대돼요.
```

**왜 이 답변이 좋은가**: 시각적 이미지("그래프", "로그") + 직무 만족도의 원천 + Allganize 연결

---

## 톤 가이드: 자연스러움의 비결

### Do

- **"~거든요"** 체 사용 → 대화체가 자연스러워짐
- **웃음이나 감정 표현** → "(웃음)", "좀 웃길 수도 있는데"
- **구체적 숫자** → "A4 3장", "30분에서 5분", "60%"
- **솔직한 자기 인식** → "참을 수가 없어요", "중독성이 있어요"

### Don't

- "저는 항상 열정적으로 일합니다" → 추상적, 믿기 어려움
- "DevOps에 열정이 있습니다" → 면접용 답변 느낌
- "최신 기술 트렌드를 항상 팔로우합니다" → 구체성 없음
- 유행어 나열 ("GitOps, IaC, SRE에 관심이...") → 깊이 없어 보임

## 꼬리 질문 대비

### "구체적으로 요즘 공부하는 거 있어요?"

```
요즘은 두 가지를 병행하고 있어요.
하나는 Kubernetes 환경에서 GPU 워크로드를 효율적으로 스케줄링하는 방법이고,
다른 하나는 Terraform으로 멀티 클라우드 인프라를 관리하는 연습이에요.

개인 홈랩에서 미니 K8s 클러스터를 돌리면서 실험하고 있는데,
프로덕션과는 스케일이 다르지만
개념을 잡는 데는 충분하더라고요.
```

### "기술 블로그나 커뮤니티 활동은 하세요?"

```
블로그를 정기적으로 쓰지는 못하고 있지만,
학습한 내용을 노션에 정리하는 습관은 있어요.
향후에는 이걸 정리해서 기술 블로그로 공개하고 싶은 계획은 있습니다.

커뮤니티는 KANS(Korea Kubernetes Network Study)나
CloudNet@ 스터디 같은 곳을 팔로우하면서
다른 엔지니어들의 경험을 참고하고 있어요.
```

### "일과 삶의 균형은 어떻게 보세요?"

```
중요하다고 생각해요.
다만 저한테는 기술 공부가 완전한 "일"은 아니에요.
퇴근 후에 Terraform 코드 만지는 게
다른 사람이 게임하는 거랑 비슷한 느낌이에요.
물론 번아웃 안 오게 쉴 때는 쉬지만요.
```
