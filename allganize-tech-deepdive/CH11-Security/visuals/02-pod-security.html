Privileged (특권)
Unrestricted / 모든 것 허용. 시스템 워크로드용
hostNetwork: true | hostPID: true | privileged: true | ALL capabilities | hostPath volumes
호스트 네트워크, PID, 특권 컨테이너, 모든 볼륨 마운트 가능
Baseline (기본)
Known privilege escalations blocked / 알려진 권한 상승 차단
No hostNetwork | No hostPID | No privileged | Limited capabilities | No hostPath
hostNetwork/PID 금지, 특권 모드 금지, 제한된 capabilities, hostPath 금지
Restricted (제한)
Maximum hardening / 최대 보안 강화. 프로덕션 권장
runAsNonRoot: true | drop ALL caps | readOnlyRootFilesystem | seccompProfile: RuntimeDefault | No privilege escalation
비 root 실행 필수, 모든 capabilities 제거, 읽기전용 파일시스템, seccomp 프로필 필수
▼ 보안 강화
▼ 최대 보안
runAsNonRoot:true | 컨테이너를 root(UID 0)로 실행하지 않도록 강제. 위반 시 Pod 시작 실패
runAsUser:1000 | 컨테이너 프로세스의 UID를 지정. 0이 아닌 값 사용 권장
readOnlyRootFilesystem:true | 컨테이너의 루트 파일시스템을 읽기 전용으로 설정. 악성코드 쓰기 방지
allowPrivilegeEscalation:false | 프로세스가 부모보다 높은 권한을 얻지 못하게 함. setuid 바이너리 차단
capabilities.drop:["ALL"] | 모든 Linux capabilities 제거. 필요한 것만 add로 추가 (최소 권한)
seccompProfile.type:RuntimeDefault | 런타임 기본 seccomp 프로필 적용. 위험한 시스템 콜 차단
seLinuxOptions:{level: "s0:c123"} | SELinux 컨텍스트 설정. MCS/MLS 기반 격리 강화
procMount:Default | /proc 마운트 방식. Default는 런타임이 마스킹 처리 (정보 노출 방지)

❌ Before: Insecure Pod (보안 미적용)

🔴root (UID 0)으로 실행
🔴privileged: true - 호스트 커널 전체 접근
🔴모든 capabilities 보유
🔴writable root filesystem
🔴hostNetwork: true
🔴hostPID: true
🔴No seccomp profile
🔴allowPrivilegeEscalation: true
spec:
  containers:
  - name: app
    image: myapp:latest
    securityContext:
      privileged: true
  hostNetwork: true
  hostPID: true

✅ After: Hardened Pod (보안 강화)

🟢runAsNonRoot: true (UID 1000)
🟢privileged: false
🟢capabilities: drop ALL
🟢readOnlyRootFilesystem: true
🟢hostNetwork: false
🟢hostPID: false
🟢seccomp: RuntimeDefault
🟢allowPrivilegeEscalation: false
spec:
  securityContext:
    runAsNonRoot: true
    runAsUser: 1000
    seccompProfile:
      type: RuntimeDefault
  containers:
  - securityContext:
      readOnlyRootFilesystem: true
      allowPrivilegeEscalation: false
      capabilities:
        drop: ["ALL"]
