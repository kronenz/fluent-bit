2. 네트워크 계층별 점검 (OSI Layer Inspection)
L7

Application Layer

HTTP 상태코드, 헤더, 응답 본문 확인. API 엔드포인트 점검.

curl -vhttpie
L6

Presentation (TLS)

TLS 핸드셰이크, 인증서 유효성, 암호화 스위트 확인.

openssl s_clientssllabs
L5

Session Layer

세션 상태, Keep-Alive, WebSocket 연결 유지 확인.

ss -tnpnetstat
L4

Transport (TCP/UDP)

포트 오픈 여부, TCP 상태, 연결 타임아웃 확인.

nc -zvtcpdump
L3

Network (IP)

IP 라우팅, 방화벽 규칙, 패킷 경로 확인.

tracerouteping
L2

Data Link (Ethernet)

ARP 테이블, MAC 주소, VLAN 설정 확인.

arp -aip link
L1

Physical

케이블 연결, NIC 상태, 링크 속도 확인.

ethtoolip link
3. 주요 트러블슈팅 명령어

DNS dig / nslookup

dig +short api.example.com dig @8.8.8.8 api.example.com A dig api.example.com +trace

DNS 레코드 조회. +trace로 전체 해석 경로 추적. @으로 특정 DNS 서버 지정.

Connection curl -v

curl -vvv https://api.example.com curl -w "%{time_connect}/%{time_total}" url curl -k --resolve host:443:1.2.3.4 url

상세 연결 정보. -w로 타이밍 측정. -k로 인증서 검증 스킵. --resolve로 DNS 우회.

TLS openssl

openssl s_client -connect host:443 openssl x509 -in cert.pem -noout -text openssl s_client -showcerts -servername host

TLS 연결 테스트, 인증서 상세 정보, 인증서 체인 확인. -servername으로 SNI 지정.

Network tcpdump / traceroute

tcpdump -i eth0 port 443 -nn traceroute -T -p 443 host ss -tnlp | grep :443 mtr host

패킷 캡처, 경로 추적, 소켓 상태 확인. mtr은 traceroute+ping 결합.
