DNS Resolution 흐름

Recursive DNS Query와 K8s CoreDNS 동작 원리

1. 재귀적 DNS 질의 (Recursive DNS Resolution)

Query: www.allganize.ai 의 IP 주소를 찾는 과정

💻 Client 클라이언트
🔍 Resolver 재귀 리졸버 (ISP) Cache TTL: 300s
🌏 Root DNS . (13개 루트 서버) TTL: 48h
🌎 .ai TLD Top-Level Domain TTL: 24h
🔐 Authoritative allganize.ai NS TTL: 300s
1. Client -> Resolver
2. Resolver -> Root
3. Root: ".ai는 여기로"
4. Resolver -> .ai TLD
5. TLD: "allganize.ai NS"
6. Resolver -> Auth NS
7. Auth: "A 1.2.3.4"
8. Resolver -> Client
2. K8s CoreDNS와 ndots 설정
📦 Pod my-app pod
🔍 CoreDNS kube-system
ndots:5 Search Domain 확장 Query: api-server (dots < 5)
1. api-server.default.svc.cluster.local
2. api-server.svc.cluster.local
3. api-server.cluster.local
4. api-server. (absolute)
📄 Search Domains .svc.cluster.local
🌐 Upstream DNS 외부 DNS 서버
✅ 10.96.0.100 ClusterIP 반환
ndots 설정

기본값 ndots:5. 쿼리의 dot 수가 ndots 미만이면 search domain을 먼저 시도. FQDN은 끝에 . 붙여 바로 질의.

TTL 캐싱

각 DNS 서버가 TTL(Time To Live)만큼 결과를 캐시. TTL 만료 전 동일 쿼리는 캐시에서 즉시 응답.

CoreDNS Plugins

kubernetes 플러그인이 Service/Pod DNS 자동 생성. forward로 외부 쿼리 전달.

DNS Record Types

A (IPv4), AAAA (IPv6), CNAME (별칭), SRV (서비스 포트), MX (메일), TXT (검증)
