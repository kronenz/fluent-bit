TIME_WAIT 필요성

마지막 ACK 손실 시 재전송 대기. 이전 연결의 지연 패킷이 새 연결에 영향주는 것을 방지. 기본 2*MSL = 60s

Connection Pooling 장점

3-way handshake 오버헤드 제거. TIME_WAIT 상태 감소. TLS 핸드셰이크 재사용으로 지연 시간 대폭 절감.

tcp_tw_reuse

net.ipv4.tcp_tw_reuse=1 설정으로 TIME_WAIT 소켓을 새 연결에 재사용 가능. 서버 측에서 유용.

Keep-Alive

HTTP Keep-Alive로 TCP 연결 유지. Connection: keep-alive 헤더. HTTP/2는 기본 활성화.
